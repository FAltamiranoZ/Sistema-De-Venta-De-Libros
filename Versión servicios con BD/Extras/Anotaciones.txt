El WSDL para poder usar el Composite Application (CA) en el estresador es: http://localhost:9080/VentaLibrosWSDLService/VentaLibrosWSDLPort?WSDL
Por alguna razón, el CA se hace undeploy cada que se reinicia el servidor, recordar siempre darle deploy
Por alguna razon, no puedo lanzar dos veces un SQLException en un web Service, así que si quiero usar más de uno, reemplazarlo con un try-catch
Si cambio el funcionamiento dentro de un web service, solo tengo que darle clean and build al pojo para que funcione, pero si cambio la estructura del web service (Parámetros, return type, etc) entonces además le debo de dar refresh
El Binding para el test del CS es el casaBinding, no el que dice VentaLibrosWSDLBinding
Si se modifica algun webService (en cuanto a estructura del webService, como un nuevo método, cambio en los parámetros, etc. No hay problema con solo la funcionalidad dentro de los métodos), el BPEL no tiene manera de actualizarlo, por lo que se tiene que hacer es: primero, eliminar los elementos relacionados con el webService, luego eliminar los objetos, luego eliminar el servicio a la derecha del BPEL, luego eliminar el wrapper, luego el wsdl, luego las referencias y por último, el texto en el source del bpel.
No esta definida la excepción en la que una conexión se corta por timeout, por alguna extraña y necia razón usa la misma que si tu hubieras cortado la conexión a mano, tener cuidado con eso.
